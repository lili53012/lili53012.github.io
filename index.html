IT Equipment Lifecycle Management System

Internship in ST Asset Managment
NYP internship attachment
IT Equipment Lifecycle Management System
Descriptions

A website project that I have done during my internship in ST Asset Management under the IT department. I was tasked to build an system called IT equipment Lifecycle Management System that helps to manage the IT equipments so as to reduce manual work in the business process and improve the efficiency of the processes. It Helps to monitor the lifecycle of the equipment systems. It includes different kind of Assets (Laptop, desktop, monitor, smartphone, printer, token and miscellaneous.)

It includes different modules such as vendor management, staff management, software management, asset management (Laptop, desktop, monitor, smartphone, printer, token and miscellaneous.), ownership management. Through the development of this web application, I had exposure to a lot of learning opportunities on how to design and develop a system from scratch, initially I was only provided with basic CRUD skeletons with no user interface and partial database design and class diagram design and partial functionalities.

As I started to work on the project, before I can really make any changes to the system. I must first understand the business workflow of different modules/functions. For example, for the part under vendor management, I was tasked to key in all the vendor related data into excel and, I had to check and key in every invoice and purchase order and also if there is any data that not capture by the system eg a signature. I would have to note it down and discuss with IT teams and then followed by update the database diagram, class diagram, UI and functionalities accordingly. Another example like staff management, I would have to key in the staff information and went through same process of checking if there was anything not captured by the system.

Moreover, I had also involved in helping in the migrating of data into the system. Basically, different modules required different ways to collect data. Like what I have mentioned previous, for vendor I have to key in all the purchase orders and invoices. Finally, I migrate the data into the system through the excel data file and using of SQL management studio. It was really a good experience and I really gain a lot and started to understand about how we can actually transform manual workflows into applications.

ACI - Trainee Management System (TMS)

FYP in Nanyang Polytechnic (Industry Project)
ACI - Trainee Management System (TMS)
Descriptions

It was an industry project partner with Asian Culinary Institute(ACI) , I was tasked to work on their Trainee Management System(TMS) to build a function for the staff to programmatically migrate their manually handled excel data into the system. In order for them to better understand how the data migration works, I had also prepare an excel instructions sheets in the exported excel. The instruction sheets will contain database schema information as well as example on the data that they can key in. which help the user to understand how to input data in the files and upload into the system.

Besides the migrating of data, I had also allow them to export the data out and it is according to the different roles of the user. They had to select the department that they want to export the data out. For example, if is front desk staff they should only able to see tables that is relevant to them. And of course data encryptions and decryption is also applied on those sensitive data.

Moreover, the excel data file that is exported out is also useful when they want to do data analytics, they can just open the excel file in the Tableanu. I had actually tried to build dashboards as an example of how they can analyse data with the file. It was really a great learning opportunities.

Technology/Language Used

Asp.Net C#, SQL Server Management Studio, Visual Studio, Excel, Tableau Software for Data Analytics

Functions/Features

Data Migration

Data Exportion

Data Analytics
